• In JavaScript, var, const, and let are all used for declaring variables, but they have some differences in terms of scope, hoisting, and mutability.

    - With the var keyword, variable names are redeclarable and reassignable.
    - With the let keyword, variable names are reassignable; we can change the value in the future in an application of a let variable.
    - With the const keyword, variable names are fixed and can't be redeclared or reassigned. When we want to make a variable fixed and never change, we use the const keyword.
    var is function-scoped, meaning it is limited in scope to the function in which it is declared or the global scope if declared outside of any function.
    - Variables declared with var can be re-declared and updated within their scope.
    - Variables declared with var are hoisted to the top of their scope.
    let is block-scoped, meaning it is limited in scope to the block ({}) in which it is declared.
    - Variables declared with let can be updated but not re-declared within their scope.
    - Variables declared with let are hoisted to the top of their block but not initialized until their declaration is evaluated.
    const is also block-scoped like let.
    - Variables declared with const cannot be re-assigned once initialized. However, for objects and arrays, their properties or elements can be modified.
    - Variables declared with const are also hoisted but not initialized until their declaration is evaluated.
    Using const is preferable when you don't intend to reassign a variable. It helps in writing more predictable and maintainable code. let is useful when you need to reassign variables. var is still used in older codebases, but let and const are preferred in modern JavaScript development due to their more predictable scoping behavior.

  • - In JavaScript, data types define the type of data that can be stored and manipulated within a program.
    - JavaScript is a loosely typed language, meaning variables can hold values of any data type without explicitly defining the type.

    - Use the typeof keyword to find a type of data or data stored in a variable.
    - console.log(typeof(varName)); OR in console: typeof varName
    - have a result of your data in a table: console.table([varName1, varName2, varName3 and so on])

  • In JavaScript, primitive data types are basic data types that are not objects and have immutable values. There are six primitive data types:

    - Number: Represents numeric values. It can be integer or floating-point.
    let num = 10;
    let floatNum = 3.14;
    - String: Represents textual data enclosed within single quotes ('') or double quotes ("").
    let str = 'Hello';
    let anotherStr = "NaseebCodeStudio";
    - Boolean: Represents a logical value indicating true or false.
    let isTrue = true;
    let isFalse = false;
    - Undefined: Represents a variable that has been declared but not assigned a value or a function that doesn't return a value. it is a data and also a value.
    let n;
    let box = undefined;
    console.log(n); // Output: undefined
    - Null: Represents the intentional absence of any value.
    let y = null;
    let selecetedProduct = null;
     - Symbol: Represents a unique identifier, used primarily as object property keys.
    const sym = Symbol('description');
    These primitive data types are used to represent simple values directly in memory, and they are immutable, meaning their values cannot be changed.

  • Types of Programming Languages:
    - there are two types of programming languages:

    1. Static programming languages
    - In static programming languages, the data type of a variable is known at compile time. Once a variable is declared, its data type cannot be changed. This means that variables must be explicitly declared with their data types before they are used.
    - Imagine you're preparing to bake cookies. In a static language, it's like having labeled jars for each ingredient before you start baking. You know exactly what goes into each jarâ€”flour, sugar, etc. Once you've labeled them, you can't change what's in the jars. Similarly, in static languages, when you declare a variable, you have to specify its type (like saying this jar is for flour), and once declared, you can't change its type later.
    Example: In languages like C, Java, or Swift, when you declare a variable, you explicitly state its data type:
    int num = 10; // 'num' is declared as an integer
    String message = "Hello"; // 'message' is declared as a string
    2. Dynamic programming languages
    - In dynamic programming languages, the data type of a variable is determined at runtime. You don't need to specify the data type when declaring a variable. Variables can hold different types of data throughout the program's execution.
    - Going back to the baking analogy, in dynamic languages, it's like having a single jar that can hold different ingredients at different times. You can start with flour, then put sugar in it later if you need to. Similarly, in dynamic languages, you can use a variable without specifying its type upfront, and you can change what's stored in it during the program's execution.
    Example: In languages like JavaScript or Python, you don't need to specify the data type when declaring a variable:
    let num = 10; // 'num' can hold a number
    let message = "Hello"; // 'message' can hold a string
    num = "World"; // Now 'num' holds a string
    In summary, the key difference is that in static languages, you need to specify the data type of variables upfront and can't change it later, while in dynamic languages, the data type is determined at runtime, and variables can hold different types of data over time.
